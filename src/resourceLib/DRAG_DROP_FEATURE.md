# 资料库拖放功能说明

## 功能概述

已为资料库添加了拖动文件/文件夹到文件夹的功能，支持在详情视图和分栏视图中使用。

## 使用方法

### 1. 拖动文件到文件夹

1. 鼠标按住文件或文件夹
2. 拖动到目标文件夹上
3. 文件夹会高亮显示（蓝色虚线边框）
4. 松开鼠标完成移动

### 2. 拖动到空白区域

- 将文件拖到列表的空白区域，文件会移动到当前文件夹的根目录

## 功能特性

### ✅ 已实现

1. **视觉反馈**
   - 拖到文件夹上时，文件夹会显示蓝色高亮和虚线边框
   - 拖放过程中显示移动光标

2. **智能验证**
   - 防止将文件夹移动到自身
   - 防止将文件夹移动到自己的子文件夹中（避免循环引用）
   - 移动成功后显示提示消息

3. **双视图支持**
   - 详情视图（列表模式）
   - 分栏视图（Finder 风格）

4. **交互优化**
   - 拖拽操作不会触发文件夹的点击导航
   - 支持拖拽多个文件（需要分别拖动）

### 📋 技术实现

1. **数据层** (`resourceLibStore.js`)
   - 新增 `moveItem()` 函数
   - 更新项目的 `parentKey` 字段
   - 自动更新 `lastEdit` 时间戳
   - 循环引用检测算法

2. **UI 层** (`ResourceLibrary.jsx`)
   - 拖拽事件处理：`onDragStart`, `onDragOver`, `onDragLeave`, `onDrop`
   - 拖拽状态管理：`dragOverFolderKey`
   - 拖拽标志：`isDraggingRef` 防止误触发点击事件

3. **样式层** (`ResourceLibrary.css`)
   - `.finder-file-row-dragover` - 详情视图拖放高亮
   - `.finder-column-item-dragover` - 分栏视图拖放高亮
   - macOS 风格的蓝色虚线边框

## 注意事项

1. **跨库移动**：当前版本仅支持在同一库内移动（个人库或个人库，组织库到组织库）
2. **批量移动**：目前需要逐个拖动文件，暂不支持批量选择后拖动
3. **撤销功能**：移动操作会立即保存到 localStorage，建议后续添加撤销功能

## 后续优化建议

2. 添加撤销/重做功能
3. 支持跨库移动（个人库 ↔ 组织库）
4. 添加拖放动画效果
5. 移动端触摸支持
